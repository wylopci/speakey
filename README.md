# 🎙️ Speakey

**按住說話,放開就打好字。**

Mac 上的 AI 語音輸入——在任何 App 的任何輸入框,用嘴巴代替鍵盤。
自動去除「嗯、啊、那個」,補好標點,還能即時翻譯。

## ✨ 功能

- **🎙 按住即說**:按住右 Option 說話,放開後 1–2 秒,文字自動出現在游標處——LINE、Email、瀏覽器、備忘錄,哪裡都行
- **🇹🇼 台語不是將就用**:市面上的語音輸入工具幾乎都不支援台語,Speakey 特別導入聯發科訓練的 Breeze-ASR-26 模型,大幅提升台語辨識準確度;還能把說出來的中文即時翻譯成道地台語漢字(實驗性)
- **🧹 AI 智慧潤飾**:不是逐字稿,是你「想表達的話」——語助詞、口吃、重複自動清乾淨,標點自動補好,還會依貼上的 App 自動切換書面/口語語氣
- **🌏 中英日韓自動偵測**:說什麼語言打什麼語言,不用手動切換
- **🔁 即時翻譯**:說中文、出英文——寫英文信、回外國客戶的神器
- **🔒 完全離線模式**:語音辨識、AI 潤稿都可以切換成本機模型執行,錄音到打字全程不連網路,注重隱私的人也能放心用
- **🧠 越用越準**:在你的電腦裡學習你的用語,自動建議常用詞;「聽錯的 => 正確的」修正規則根治頑固錯字
- **🕶 隱私設計**:無帳號、無追蹤、無自家伺服器;使用紀錄只存在你的電腦,可關可刪

## 📥 下載安裝

1. 到 **[Releases](https://github.com/wylopci/speakey/releases/latest)** 下載對應版本:

   | 你的 Mac( → 關於這台 Mac) | 下載 |
   |---|---|
   | 晶片:Apple M1 / M2 / M3 / M4 | `Speakey-AppleSilicon.dmg` |
   | 處理器:Intel Core i5 / i7 / i9 | `Speakey-Intel.dmg` |

2. **雙擊 `.dmg`**,把 Speakey 圖示拖進「應用程式」
3. 打開「應用程式」**雙擊開啟**(已通過 Apple 公證,不會有任何警告)
4. 照畫面指示:貼上免費的 Groq API Key(App 會帶你申請)→ 開啟權限 → 開始用

詳細步驟見 DMG 內的互動式《安裝說明》(支援繁中/English/日本語/한국어,依系統語言自動切換)。

## 🔑 費用

**Speakey 完全免費。** 語音辨識使用你自己申請的 [Groq](https://console.groq.com) 免費額度,日常使用綽綽有餘——你的 Key、你的資料、你作主。

## 🛡 隱私與安全

**會經過雲端的部分(功能必要)**
- 你說的話與辨識出的文字,會加密傳送到 Groq 做語音辨識與 AI 潤飾/翻譯
- 預設模型是 OpenAI 開源釋出的 `whisper-large-v3`(語音辨識)與 `gpt-oss-120b`
  (文字潤飾),運算皆在 Groq(美國公司)的伺服器上執行——**預設不使用任何中國廠商的 AI 模型**
- 選單可自行切換其他模型(每個選項都標注開發商與國別,含中國廠商模型可選);自行切換後即以你選的模型為準
- Groq 官方聲明不保留、不用於訓練

**100% 留在你電腦、永不上傳的部分**
- 你的 Groq API Key(存在你自己的使用者目錄)
- 使用紀錄、常用詞學習、自定詞庫、修正對照表
- 執行 Log

**開發者這邊**
- Speakey 沒有自己的伺服器,程式碼裡沒有任何上傳邏輯
- 資料只在「你的電腦」和「你自己申請 Key 連的 Groq」之間流動,開發者看不到、也無法收集任何使用內容
- 鍵盤監聽只判斷「是否按下快捷鍵」,不記錄任何按鍵內容;密碼欄位受 macOS Secure Input 保護,語音輸入天生無法誤入

## 💻 系統需求

- macOS 12(Monterey)以上;「開機自動啟動」需 macOS 13+
- Apple Silicon 或 Intel 晶片(請下載對應版本)
- 需要網路(語音辨識為雲端)

## 🗒 版本更新紀錄

### 3.3.0(2026-07-24)
- **潤稿依情境自動分段**:貼到 Email 或筆記類 App 時,如果內容涵蓋多個重點或話題轉折,潤稿會依原本內容的邏輯結構自動分段,不會擠成一大段連續文字;只調整分段/換行,絕不會加入原文沒提到的內容(即時通訊等情境不受影響)
- 修正潤稿安全機制的誤判:同義詞替換(例如「這禮拜」被模型改寫成「本週」)可能讓「答非所問」的偵測機制誤判、白白丟棄正常的潤稿結果,調整判斷門檻後已修正

### 3.2.1(2026-07-24)
- **修正文字偶爾無法自動貼上的問題**:新增對 macOS「自動化」權限(控制 System Events 送出 Cmd+V 用,跟輔助使用是獨立的權限)的主動偵測,被拒絕時會導引到系統設定;若貼上失敗,也不會再自動把剪貼簿還原掉,結果會保留久一點並提示可手動 Cmd+V,不會讓辨識結果整個消失

### 3.2.0(2026-07-23)
- **新增「台語漢字(實驗性)」輸出語言**:潤稿完的中文可再翻譯成台語漢字,使用本機 SARC-Taigi-LLM 模型(基於 Google Gemma-3,由陽明交通大學語音AI研究中心微調釋出),完全離線執行,僅 Apple Silicon 提供。經多輪測試翻譯品質穩定,但仍標示實驗性,建議謹慎使用
- 條列式格式(1. 2. 3.)在台語漢字翻譯時會正確保留數字編號,不會被誤翻成中文數字
- **本機模型管理選單整理**:「移除本機語音辨識模型」「移除本機 TAIDE 模型」「移除台語漢字翻譯模型」「顯示模型儲存位置」「檢查本機模型更新」收攏成一個「本機模型管理」子選單,「進階設定」不再被一堆項目塞滿
- 辨識語言選「台語」時,若目前是線上(Groq)辨識,Apple Silicon 會要求下載/切換離線模型才能繼續切換(離線模型準確度明顯較好);Intel 因為沒有離線模型可用,維持提醒但不強制
- 修正本機語音辨識偶爾對短句產生幻覺(講很短的話卻辨識出無關的一兩個字)的問題,加入重試機制

### 3.1.1(2026-07-23)
- **修正首次設定精靈的選項互斥錯誤**:「語音辨識」與「潤稿」兩組引擎選項原本共用同一個容器,導致點選其中一組會連帶清掉另一組的選取狀態。現在兩組各自獨立,互不影響

### 3.1.0(2026-07-23)
- **新增首次啟動設定精靈**——全新安裝時會引導完成三件事:語音辨識/潤稿要用線上(Groq)、線下(本機)還是兩者都要(含下載空間揭露);預設辨識語言(選到台語會提示線上準確度較低,建議順便下載離線模型);Groq API Key 申請與設定。跑完之後所有選項依然可在「進階設定」隨時修改
- 已安裝且已設定過 API Key 的使用者升版不受影響,不會看到這個精靈

### 3.0.0(2026-07-23)
- **本機語音辨識雙模型架構**:中文/台語自動改用聯發科 Breeze-ASR-26,實測辨識準確率明顯優於通用模型;英/日/韓與自動偵測則維持使用原本的 whisper-large-v3-turbo,兩者依偵測到的語言自動切換,不需手動選擇
- 拿掉「台語(實驗性)」的「(實驗性)」字樣——雙模型架構上線後實測已可正式使用
- **新增本機模型版本更新檢查**:TAIDE / Breeze-ASR-26 / 通用 Whisper 皆會定期比對來源是否有新版,偵測到更新會提示並可一鍵重新下載,選單項目也會即時標註「(有更新)」
- 「進階設定」新增「顯示模型儲存位置」,一鍵在 Finder 開啟模型資料夾,不用自己找路徑
- 修正:移除本機模型失敗時(權限問題、檔案佔用等)畫面上完全沒有提示,現在會顯示「移除失敗」
- EULA(授權條款)補上本機模型授權揭露:TAIDE、Breeze-ASR-26、Whisper 各自的發布單位與授權條款

### 2.9.3(2026-07-22)

### 2.9.2(2026-07-22)

### 2.9.1(2026-07-22)
- **新增條列式格式化**:口述時明確用「第一點/第二點/首先/其次」這類序列詞列點,AI 潤稿會自動格式化成「1. 2. 3.」的條列式;如果口誤跳號或重複編號,也會依講話順序自動重新排序。每一點的實際內容一律照實保留,AI 不會幫你調解前後矛盾或猜測正確點數
- **修正**:移除本機模型(TAIDE / 語音辨識)後,選單裡「移除本機模型」的項目沒有即時消失,要重啟 App 才會更新。現在每次打開「進階設定」都會重新檢查

### 2.9.0(2026-07-21)
- **新增本機語音辨識**:內建 whisper.cpp 引擎(不透過 Groq),模型檔約 550MB,只需下載一次。搭配本機 TAIDE 潤稿,可達到「錄音→辨識→潤稿」全程離線、連網路都不用連
- **僅支援 Apple Silicon**(M 系列晶片);Intel Mac 上此選項不會出現,原因與 TAIDE 本機潤稿相同——純 CPU 推理速度過慢不符實用門檻
- 修正:本機語音辨識設「自動偵測語言」時,講短句子(1~2 秒)容易被誤判成需要翻譯,結果整段變成英文。現在會先偵測語言、確認後再轉錄,短句也能正確辨識原本的語言
- 新增「移除本機語音辨識模型」管理選項(釋放約 550MB)
- EULA 補充揭露本機語音辨識模型的來源(OpenAI 開源 Whisper 權重,經 ggerganov/whisper.cpp 釋出)

### 2.8.2(2026-07-21)
- **加強修正**:2.8.1 的修正在部分情況下仍可能不夠穩定,這次改用更根本的偵測方式——檢查潤稿結果跟原文的「雙字詞重疊率」,只要 AI 把內容整個換成不相干的回答(不管有沒有使用特定關鍵字或條列格式),都會被偵測到並自動退回原始逐字稿。已用真實發生過的案例驗證修正生效

### 2.8.1(2026-07-21)
